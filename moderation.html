Moderation

Use Supabase for admin actions.

A secure admin dashboard cannot live on GitHub Pages alone without real authentication and protected server-side permissions.

Safest beginner option

For now, moderate in Supabase while logged in as the project owner. The public site uses only the anon key, so it can safely submit, report, comfort, and read visible posts, but it cannot safely hide, unhide, or delete posts from a public admin page.

In Supabase, open Table Editor, choose feelings, and use the columns hidden, approved, reported_count, and comfort_count.

  • Newest first: sort created_at descending.
  • Most reported: sort reported_count descending.
  • Hide: set hidden to true.
  • Unhide: set hidden to false.
  • Delete: use only when absolutely needed.

Auto-hide is already handled in the database: when a post reaches 3 unique reports, hidden is set to true.
